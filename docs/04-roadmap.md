## M0 · 기반 (진행 중)

목표를 정하고, 콘텐츠 규칙을 확정하고, 지면을 정한다. **코드는 최소한만.**

- [x] 설계 문서 — `docs/00`~`05`
- [x] **데이터 스키마 v2 확정** — `src/types/entry.ts`. `audio.us` 필수, `Collocation`, `StudyRecord`/`StudyState`, `EntryId` 참조, 뜻갈래 단위 `tags`, `editorialStatus`, 버전 필드
- [x] **검증기 계약** — [05-validation-contract](05-validation-contract.md). 토큰화 명세, 순환 탐지 방식, 검사 항목 V01~V16
- [x] 유효 샘플과 위반 fixture 분리 — `data/entries.sample.json` / `data/entries.invalid.json`
- [x] 지면 조판 시안 — `prototype/entry.html`, 미국식 기준 (ADR-005)
- [x] 예문 길이 규칙 재검토 — 8~12 → **6~12단어**
- [x] **교육부 어휘 목록 원문 확인** — 고시 제2022-33호 [별책 14] [별표 3]. 초등 800단어, 학습 어휘 상한 600단어. 추출본 `data/moe-vocabulary.json`
- [ ] 아이의 기기·입력 방식 확인 (남은 미해결 질문)
- [ ] **어휘 목록 검수** — 추출본 819개를 원문 800개와 대조. 표제어 입력 전에 끝내야 한다
- [ ] **정의 어휘 1,000단어 목록 확정** — `data/defining-vocabulary.json`. 초등 800 + 정의용 기능어 200
- [ ] 불규칙 굴절 매핑 — `data/inflections.json`
- [ ] 면제 부류 목록 — `data/exempt-classes.json`. 단위명·약어·호칭은 소문자라 명시 필요
- [ ] **검증기 구현** — V01~V16. `entries.sample.json`은 전 항목 통과, `entries.invalid.json`은 각 표제어가 `_expect`의 코드만 내야 한다

**완료 기준**: 표제어 20개를 [편집 체크리스트](01-content-policy.md#편집-체크리스트) 전 항목 통과로 입력할 수 있고, 검증기가 fixture의 위반 15건을 정확히 잡아낸다.

> **M0에서 의도적으로 미룬 것.** 아래는 코드만 고치면 되돌릴 수 있으므로 해당 마일스톤에서 정한다 — Leitner 상태 전이표·오답 보기 완화 순서(M2), 서비스 워커 캐시 전략(M1), 학습 기록 가져오기 검증(M2).
> **되돌릴 수 없는 것만 M0에서 고정한다**는 원칙이다. 명세를 늘리다 표제어를 한 개도 못 쓰는 것이 이 단계의 진짜 위험이다.

## M1 · 사전 — P1 + P2

**아이가 모르는 단어를 찾을 수 있게 한다.** 학습 기능은 아직 없다.

- [ ] 표제어 200개 입력 (아이가 실제로 물어본 단어 우선)
- [ ] **TTS 엔진과 재배포 조건 확정** → ADR-004에 기입
- [ ] 오디오 생성 파이프라인 + V11 파일 실재 검사 활성화
- [ ] 앱 골격 — Vite + React + TS, PWA, 오프라인
- [ ] **서비스 워커 캐시 갱신 전략** — `contentVersion` 기준 (ADR-008)
- [ ] 검색 — 오타 관용 (`elefant` → `elephant`)
- [ ] 표제어 화면 — 시안 그대로. 정의 내 `[[ ]]` 링크로 다른 표제어 이동
- [ ] 학년 설정 → 뜻갈래 노출 제어

**완료 기준**:
- **관찰 절차**: 아이가 모르는 단어 **5~10개**를 앱에서 혼자 찾게 하고 옆에서 보기만 한다. 힌트를 주지 않는다. 되묻는 표제어가 나오면 **그 표제어가 편집 실패**이므로 고친 뒤 다시 관찰한다
- 네트워크를 끊고 전 기능이 동작한다

## M2 · 반복 — P3

**찾은 단어를 잊지 않게 한다.** 설계는 [02-learning-design](02-learning-design.md).

- [ ] 표제어 500개까지 확대
- [ ] **상태 전이표 확정** — 신규 카드 생성, 당일 오답 재출제, 5번 박스 이후, 밀린 복습 선별, 학년 변경 시 처리
- [ ] Leitner 5박스 + 학습 기록 (IndexedDB). `dueOn`은 로컬 날짜 문자열 (ADR-007)
- [ ] 하루 세션 — 신규 5 + 복습 최대 15, 명확한 끝
- [ ] 문제 유형 3종 — 뜻 고르기 / 소리 듣고 고르기 / 철자 채우기
- [ ] **오답 보기 생성** — 보기 부족 시 완화 순서, 중복 한국어 뜻 제외, 다의어·굴절형 정답 판정
- [ ] 오답 화면 — 정답 + 소리 + 예문. **재생 실패해도 "다시 듣기" 버튼과 텍스트 피드백 보장**
- [ ] 찾아본 단어 자동 등록 — **어느 뜻갈래를 등록할지, 단순 탐색도 등록할지 함께 정한다**
- [ ] 학습 기록 내보내기·불러오기 (ADR-002). **깨진 파일이 기존 기록을 덮어쓰지 않을 것**
- [ ] **4번 박스 첫 복습 정답률을 설정 화면에 표시** — 아래 완료 기준의 측정 수단

**완료 기준**:
- 아이가 시키지 않아도 하루 학습을 시작한다
- **4번 박스(16일 간격) 첫 복습 정답률 80% 이상.** 별도 시험을 치르지 않는다 — 16일 전 학습한 것을 오늘 묻는 게 정확히 이 지표이고, 앱이 매일 자동으로 갱신한다
