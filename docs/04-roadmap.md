## M1 · 사전 — P1 + P2

**아이가 모르는 단어를 찾을 수 있게 한다.** 학습 기능은 아직 없다.

- [x] **앱 골격** — Vite + React 19 + TS. `npm run build` 통과. 의존성 설치 후 `npm run dev`
- [x] **PWA·오프라인** — vite-plugin-pwa. 앱 셸 + 번들 데이터 precache(216 KiB), autoUpdate
- [x] **검색 — 오타 관용** — 편집거리 기반. `skool→school`, `windo→window`, `bred→bread` 확인. 한국어 대응어 검색도 됨(`고양이→cat`)
- [x] **표제어 화면** — 프로토타입 조판 그대로. 음절점·US 발음·품사 색상·굴절형·연어·관련어
- [x] 정의 내 `[[ ]]` 링크로 다른 표제어 이동 — `DefinitionText` 구현
- [x] 학년 설정 → 뜻갈래 노출 제어. localStorage 저장. 다크/라이트 테마 토글
- [x] **저작 소스 + 생성 파이프라인** — `data/source/entries.src.json` → `build.py` → `entries.json`. IPA는 CMUdict, 음절은 pyphen에서 생성. 판단이 필요한 정의·예문·한국어만 사람이 저작
- [x] **표제어 입력 — 200개 완성.** 검증기 위반 0건, 예문 목록 밖 어휘 1.8%(기준 10%). 발음 오디오 200개
- [x] **TTS 엔진·라이선스 확정** — Piper (MIT) + en_US-lessac-medium. 오프라인·재배포 자유. ADR-004에 기입
- [x] **오디오 생성 파이프라인** — `tools/audio/generate.py`. 132개 mp3(459 KB) 생성·커밋. V11 파일 검사 활성(`npm run validate:audio`)
- [x] **오디오 앱 통합** — 발음 버튼이 mp3 재생, 실패 시 Web Speech 대체. PWA precache에 mp3 포함
- [ ] **서비스 워커 캐시 갱신 전략 세부** — `contentVersion` 기준 갱신 알림 (ADR-008). 현재는 autoUpdate로 시작

**완료 기준**:
- **관찰 절차**: 아이가 모르는 단어 **5~10개**를 앱에서 혼자 찾게 하고 옆에서 보기만 한다. 힌트를 주지 않는다. 되묻는 표제어가 나오면 **그 표제어가 편집 실패**이므로 고친 뒤 다시 관찰한다
- 네트워크를 끊고 전 기능이 동작한다 — 서비스 워커가 mp3까지 precache. 표제어 200개 도달 후 실측 예정

> **M1 콘텐츠 완성 (2026-07-22).** 표제어 200개, 발음 오디오 200개(1 MB, PWA precache 207 entries). 헤드리스 Chromium으로 200개 렌더·영어/한국어 검색·발음 fetch 확인, 콘솔 에러 0.
> **저작 소스 + 생성 파이프라인이 검증됐다.** 68개 추가는 소스 68행 저작 → `npm run entries && npm run audio && npm test`로 끝났다. IPA·음절·오디오·검증 전부 자동. M2·M3 콘텐츠도 같은 방식으로 확장한다.

> **앱 골격 검증 (2026-07-22).** 헤드리스 Chromium으로 20개 표제어 렌더·검색·오타 관용·한국어 검색·학년 필터·다크 테마를 구동 확인. 콘솔 에러 0건. MCP가 chrome 채널 고정이라 캐시된 chromium 바이너리로 직접 구동했다.

## M2 · 반복 — P3

**찾은 단어를 잊지 않게 한다.** 설계는 [02-learning-design](02-learning-design.md).

- [ ] 표제어 500개까지 확대
- [ ] **상태 전이표 확정** — 신규 카드 생성, 당일 오답 재출제, 5번 박스 이후, 밀린 복습 선별, 학년 변경 시 처리
- [ ] Leitner 5박스 + 학습 기록 (IndexedDB). `dueOn`은 로컬 날짜 문자열 (ADR-007)
- [ ] 하루 세션 — 신규 5 + 복습 최대 15, 명확한 끝
- [ ] 문제 유형 3종 — 뜻 고르기 / 소리 듣고 고르기 / 철자 채우기
- [ ] **오답 보기 생성** — 보기 부족 시 완화 순서, 중복 한국어 뜻 제외, 다의어·굴절형 정답 판정
- [ ] 오답 화면 — 정답 + 소리 + 예문. **재생 실패해도 "다시 듣기" 버튼과 텍스트 피드백 보장**
- [ ] 찾아본 단어 자동 등록 — **어느 뜻갈래를 등록할지, 단순 탐색도 등록할지 함께 정한다**
- [ ] 학습 기록 내보내기·불러오기 (ADR-002). **깨진 파일이 기존 기록을 덮어쓰지 않을 것**
- [ ] **4번 박스 첫 복습 정답률을 설정 화면에 표시** — 아래 완료 기준의 측정 수단

**완료 기준**:
- 아이가 시키지 않아도 하루 학습을 시작한다
- **4번 박스(16일 간격) 첫 복습 정답률 80% 이상.** 별도 시험을 치르지 않는다 — 16일 전 학습한 것을 오늘 묻는 게 정확히 이 지표이고, 앱이 매일 자동으로 갱신한다

## 지금 할 일

**M0가 사실상 끝났다.** 다음은 M1(사전) 착수이며, 그 전에 미해결 질문(아이의 기기·입력 방식)을 확인해야 한다. M1의 첫 작업은 표제어를 200개로 늘리고 오디오 파이프라인을 세우는 것이다.

> **화이트리스트는 두 번 다 실제로 작동했다.**
> - 정의 어휘를 만들자마자 샘플에 걸었더니 `trunk`, `serious`, `instrument`가 걸렸다. `instrument`는 `a piano, a drum, or a guitar`로 바꾸는 편이 초등 정의로 더 낫다는 것도 드러났다
> - 20개 입력 때는 `window` 정의의 `lets`가 걸렸다. `let`은 놀랍게도 초등 800에 없다. 정의를 고쳤다
>
> 두 경우 모두 사람 눈으로는 놓쳤을 단어를 검증기가 잡았다. 제약이 더 나은 정의를 강제한다.
